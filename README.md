# 六扇门 V8 · 足彩盘口规则引擎 + 三明治法交易决策流程

## 概述

六扇门是一个基于赔率和盘口数据的足彩分析系统，不预测比赛结果，而是通过三层结构帮助识别市场中的交易机会：

1. **引擎层** — 方向投票 + 共识验证 + 价格发现
2. **庄家平衡** — 工具联动 + 赔付结构分析
3. **三明治法** — 合拍度 + 破绽 + 选标的

## 三层架构

```
赔率数据 → 引擎 → 庄家平衡 + 价格确认 → 三明治法 → 决策
```

### 第一层：引擎（football_analyzer.py）

Pin / Bet365 / Crown 三家数据，六步流程：

| 步骤 | 名称 | 说明 |
|------|------|------|
| ① | 方向 | 三家亚盘投票，有变动看变盘方向，无变动→PASS |
| ② | 共识 | 欧赔3家投票 + 平局翻转 + 收敛度 |
| ③ | 价格发现 | 价差变化判断→偏贵/合理/偏便宜 |
| ④ | 风控否决 | 三条铁律过滤器（R3/R4/R5） |
| ⑤ | 庄家反证 | 线位同步、市场成熟、欧赔匹配、反方理由 |
| ⑥ | 最终决策 | EXECUTE / WATCHLIST / PASS |

### 第二层：三明治法（TRADING_FLOW.md）

引擎输出后的人工决策流程：

1. **看合拍度** — 庄家最不舒服的结果 = 我的方向？
2. **找破绽** — O/U 价量背离、滞后定价
3. **选标的** — AH 还是 O/U？哪家公司？什么线位？

### 第三层：你

最终决定权。引擎和流程只提供信息，不下注。

## 风控系统（三条铁律）

| 规则 | 条件 | 动作 | 含义 |
|------|------|------|------|
| R4 结构冲突 | Pin ≠ 365 | PASS | 专业市场 vs 大众市场分裂 |
| R5 亚洲盘口 | Pin=365 但 Crown 反向 | PASS | 亚洲盘口不给确认 |
| R3 价格否决 | 偏贵 + 方向弱 | PASS | 又贵又没人买 |

## 公司角色

| 公司 | 角色 | 市场定位 |
|------|------|---------|
| Pinnacle | 价格锚 + 方向投票 + 欧赔验证 | 专业盘（低水） |
| Bet365 | 方向投票 + 欧赔验证 | 大众市场 |
| Crown (皇冠/singbet) | 方向投票 + 欧赔验证 | 亚洲盘口 |

## 使用方式

```bash
python3 football_analyzer.py <数据.md>
```

数据格式要求：包含欧指、亚盘、大小球三个表格，覆盖 Pin/365/Crown 三家公司。

详细格式参考 SKILL.md 或 TRADING_FLOW.md。
