# 市场决策引擎 V8

## 核心原则

不预测比赛，只做三件事：
- 依据盘口与赔率变化，推断庄家的风险管理和资金引导策略
- 识别市场是否存在价值盘口
- 输出可执行的交易决策（执行/观察/放弃）

## 核心理念

1. 市场往哪边形成共识？（Direction + Consensus）
2. 当前价格是否值得交易？（Price + Risk）
3. 是否存在合理反方解释可以推翻结论？（Bookmaker Challenge）

三者同时成立 → EXECUTE。任一无法成立 → WATCHLIST 或 PASS。

## 六层流程

```
① Direction（方向）
   Pin / Bet365 / 皇冠 三家亚盘投票
   有变动看变盘方向，无变动看静态线位
   输出：主 / 客 / 无方向（三家冲突→PASS）
       ↓
② Consensus（共识验证）
   - 欧赔支持：5家欧赔是否同向降方向方赔率
   - 平局翻转：条件A(经典) / 条件B(平赔骤降)
   - 收敛度：初盘～现盘极差变化
   输出：Strong / Neutral / Reject（Reject→PASS）
       ↓
③ Price Discovery（价格发现 v2）
   软庄天生比 Pin 低是常态，比绝对水位无意义。
   只看「软庄与 Pin 的价差从初盘到现盘变没变」：
   - 策略A：snap/curr 都与 Pin 同线位 → 比价差变化（ΔΔ）
   - 策略B：自身线位稳定（未跟Pin推盘）→ 比自身水位变化（Δ）
   - 价差拉大 → 追价；价差不变/缩小 → 正常跟随
   追价 ≥3家 → Expensive；否则 → Fair（只描述，不判断价值）
       ↓
④ Risk Filter（风控否决）
   R3 价格否决：偏贵 + 方向弱（多数未追价）→ PASS
   R4 结构冲突：Pin vs 365 方向相反 → PASS
   R5 皇冠逆多数：Pin/365同向但皇冠相反 → PASS
   ⚠️ 只有三条铁律，R1/R2/R6已移除
       ↓
⑤ Bookmaker Challenge（庄家反证）
   BC1～BC4 四道交叉检查
   输出：无异议 / 关注 / 过热
       ↓
⑥ Final Decision（最终决策前，先问一句）
   我买的价格还有没有价值？
   ⚠️ 价格已被散户追到没有优势 → 放弃或降仓
   ✅ 价格合理，没有过热追价 → 可以上
   答完这个问题，再看结论：
   EXECUTE / WATCHLIST / PASS
```

## 平局翻转条件

- **条件A**（经典）：方向方↓ + 平赔↓ + 对手方↑（任一公司触发）
- **条件B**（骤降）：平赔降≥4%的≥3家

翻转规则：**只有方向方是让球方时才翻转到受让方**；
若方向方已是受让方，则不翻转，保持方向，标记 WATCHLIST。
判断方法（看线位符号）：
- 线位<0 = 主队让球 → 主队是让球方
- 线位>0 = 客队让球 → 客队是让球方
  - dir_ah='+' 且 线位<0 → 方向主队是让球方 → 可翻转
  - dir_ah='-' 且 线位>0 → 方向客队是让球方 → 可翻转
  - 其他 → 方向已是受让方 → 不翻转

## 交易决策流程（第二层）

引擎输出结论后，参考 **[TRADING_FLOW.md](./TRADING_FLOW.md)**（三明治法）进行最终下注决策：

1. **第一层**：引擎结论 → PASS/WATCHLIST/EXECUTE 快速分类
2. **第二层**：庄家平衡 → 看合拍度 + 找破绽（O/U 价量背离、滞后定价）
3. **第三层**：价格确认 → 不看结论看数字，选最优标的（AH/O/U/哪家公司/哪个线位）

**核心原则**：AH 市场和 O/U 市场独立评估。价格偏贵只影响 AH，不波及 O/U。
