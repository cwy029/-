# 博彩交易决策流程（三明治法）

## 核心理念

```
数据 → 引擎（整理事实）→ Minis（标出信号）→ 你（从庄家角度思考→拍板）
```

三层分工：

1. **引擎** — 处理原始数据，输出方向、共识、价格、庄家平衡
2. **Minis（我）** — 整理引擎输出，标出值得注意的信号，不做庄家判断
3. **你** — 看庄家平衡数据，自己从庄家角度思考线位为什么这么走、庄家最怕什么，然后拍板

**引擎和我不替你做庄家判断。那是你的事。**

---

## 第一层：系统结论 → 快速分类

| 结论 | 行动 |
|------|------|
| 🚫 PASS | 直接扔掉，一眼不看 |
| 🔶 WATCHLIST | 进入第二层。只考虑小注或 O/U 方向。如果庄家平衡有明显破绽（如 O/U 价量背离）可以上 |
| ✅ EXECUTE | 进入第二层，正常评估 |

**原则：** PASS 是硬过滤，不纠结。WATCHLIST 不是不能买，是买之前必须有庄家平衡的额外理由。

---

## 第二层：庄家平衡 → 找信号

拿到庄家平衡输出的 4 项工具（线位/水位/欧赔/平赔 + 大小球），做两件事：

### ① 看合拍度

庄家最不舒服的结果 = 我的方向？

**平赔信号：**
- 平赔 **涨**（方向方赔率不动或降）→ 庄家怕平局 → 我买让球方方向跟庄家最不舒服的一致 → **合拍 ✅**
- 平赔 **跌**（方向方赔率涨）→ 庄家在接平局资金 → 如果买让球方要注意平局风险
- 平赔 **全线暴跌**（≥3家、跌幅10%+）→ 庄家在挡平局资金 → 方向方可能赢不了，考虑受让方或平局

**大小球信号参考：**

庄家调整大小球的工具只有两个——线位和水位。两者配合可以读出庄家倾向，但不是铁律，需要结合平赔和合拍度一起看：

| 工具组合 | 常见庄家意图 | 但要注意 |
|---------|------------|--------|
| 升盘 + 大球水涨 | 有可能在诱大 | 也可能是平衡资金，查平赔方向 |
| 升盘 + 大球水降 | 有可能真看大 | 查平赔是否配合，方向赔率是否降 |
| 升盘 + 大球水不动 | 中性 | 无倾向 |
| 退盘 + 小球水涨 | 有可能在诱小 | 同上 |
| 退盘 + 小球水降 | 有可能真看小 | 查平赔是否配合（平赔暴跌配合小球=可靠） |
| 线位不动 + 大球水降 | 资金在静悄悄进大球 | 方向赔率是否也在降 |

没有孤立的大小球信号——所有 O/U 判断必须跟平赔、方向合拍度交叉验证。

**线位与配合：**
- 深盘（-1.5+）配大球不动 → 庄家对穿盘信心不足 → 换 O/U 或降仓
- 让球方方向 + 大小球升盘 + 大球水降 → 庄家既看让球方也看进球 → 方向一致 ✅
- 线位推深 + 水位升高（让利吸筹）→ 庄家在平衡资金，不是真看好

### ② 找破绽

庄家在哪个市场留了定价错误？

- 365 还在挂旧线位，Pin/皇冠已经跑了 → **365 滞后定价 ≈ 免费钱**
- 大球水暴涨但线位推高 → **诱大，买小**
- 软庄水位明显比 Pin 高 → **买软庄线位（更安全），不追 Pin 的深线位**
- 皇冠已经退盘了但 365/威廉还没动 → **同方向价值差**
- 大小球庄家之间线位差 ≥0.50 → **买低线位那家**（更安全，逻辑一致）

---

## 第三层：价格确认 → 选标的

不看结论（偏贵/合理），看**具体数字**来决定买什么、买哪家、买哪个线位。

**核心原则：AH 市场和 O/U 市场独立评估。**
价格确认的"偏贵"只说明 AH 方向的价格已没有优势，不波及 O/U 市场。
O/U 市场的价格信号（升盘/退盘+水位变动）独立判断，不受 AH 偏贵影响。

### ① 选标的类型

| 情况 | AH 行动 | O/U 行动 |
|------|---------|----------|
| AH 偏贵（追价≥3家） | ⚠️ 降仓或放弃 | 独立评估，正常分析 |
| AH 价格合理 | ✅ 正常上 | 独立评估 |
| AH 无可比（只有Pin该线位） | 换便宜线位或 O/U | 独立评估 |
| O/U 有破绽（诱大/诱小/滞后） | — | ✅ O/U 优先 |
| 庄家平衡有诱盘信号 | — | ✅ O/U 做反方向 |
| AH 方向强但线位太深（-1.5+） | ⚠️ 降仓 | 大球代替 AH |
| 两者信号都不强 | 不买 | 不买 |

**注意：** 价格确认输出的"追价≥3家"只存在于 AH 市场（O/U 没有价差比较逻辑）。
所以"偏贵"是一个纯 AH 信号，不应对 O/U 下的注产生任何影响。

### ② 选具体产品

| 因素 | 正常仓位 | 降仓 | 放弃 |
|------|---------|------|------|
| 一层结论 | EXECUTE | WATCHLIST | PASS |
| 合拍度 | 合拍 | 有矛盾 | — |
| 破绽大小 | 明显 | 一般 | 无 |
| AH 价格 | 合理 | 偏贵 | 无可比 |
| O/U 信号 | 清晰（价量配合） | 一般 | 无信号 |
| 概率参考 | 正常 | 不确定 | 低 |

**大致分配（AH和O/U独立评估，不合并计算总仓位）：**
- EXECUTE + 合拍 + AH价格合理 → AH 15-20%
- EXECUTE + 合拍 + AH偏贵 + O/U有破绽 → 放弃AH，O/U 10-15%
- EXECUTE + 合拍 + AH偏贵 + O/U无信号 → 全部放弃
- EXECUTE + 矛盾 / WATCHLIST + 破绽明显 → 5-10%（O/U为主）
- 其他 → 0-5%

## 决策树（一张图）

```
系统结论
│
├─ PASS → ❌ 跳过
│
├─ WATCHLIST
│   └─ 庄家平衡有破绽吗？
│       ├─ 有 → 小注 O/U 方向（大球/小球），5-10%
│       └─ 无 → 跳过
│
└─ EXECUTE
    └─ 先问：价格还有没有价值？
        ├─ 被散户追到没价格（追价偏贵）→ ⚠️ 降仓或放弃
        └─ 价格合理（无过热追价）→ ✅ 可以上
            └─ 价格确认看 AH
                ├─ AH 价格合理 → AH 15-20%
                ├─ AH 偏贵 → 放弃 AH，检查 O/U 信号
                │   └─ O/U 有信号 → O/U 10-15%
                │   └─ O/U 无信号 → 放弃
                └─ AH 无可比 → 换浅档线位或 O/U
```
